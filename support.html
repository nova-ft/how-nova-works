Provisioning & deploy

Stand up a bank, then ship safely

Two operations run the lifecycle: Provisioning builds a new isolated stack from the platform masters, and Deployment ships every change per customer cloud with canary, rollback, and a human deploy-gate.

Fully isolated per bank: dedicated tenant, code, and infra — no shared blast radius.

The DR flow
🛑
Resilience gate: DR drills run on a cadence, and the gate blocks any critical-service deploy without a recent passed drill and an updated runbook. Provable recovery — not just a documented plan.

Fleet resilience — if an agent stops working

Nova FT is a fleet of agents, so resilience means the fleet degrades safely. Every change an agent makes is gated by Git + human approval, so a failed agent means paused work, not damage — nothing ships unreviewed. From there the fleet self-heals, and humans remain the ultimate failsafe.

Heartbeat
Dead-man's switch

Every agent emits a heartbeat. Miss it and the agent is presumed down — its in-flight work is frozen, not force-completed.

Master Orchestrator
Reroute & retry

Detecting a stalled agent, the orchestrator reroutes the task to a healthy peer or retries it, so the queue keeps moving.

Meta / Governance
Quarantine

An agent behaving out of policy is quarantined — isolated from live work until reviewed, so a misbehaving agent can't spread.

Human-in-the-loop
Kill-switch

A human can pause any single agent, a whole department, or the entire fleet at once — instantly, at any time.

🛑
Humans are the ultimate failsafe. Autonomy is bounded by allowlists, Git, and approval gates; the human kill-switch overrides everything above it. The fleet is designed to fail paused and safe, never runaway.
