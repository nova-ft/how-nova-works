THE COMPANY MODEL

A factory, not a consultancy.

Nova is a world-class company where the departments are AI agents and a small elite human team supervises them. Banks describe what they need in plain language; the fleet turns it into specs, code, infrastructure and running software — version-controlled at every step.

The consultancy trap

Bespoke, per customer

A consultancy re-builds from scratch for each client. Cost scales with headcount; margin erodes with every new engagement. Nothing compounds.

  • —Every bank is a fresh, hand-wired project
  • —Humans do the repetitive build work
  • —Knowledge stays in people's heads
The Nova factory

A reusable product line

Nova separates the platform (the reusable product, a fixed set of repos) from each customer instance (one bank's code, config and infra). The platform is what Nova sells; customer repos are what the product produces.

  • ✓Platform repos stay flat as customers grow
  • ✓Customer repos multiply per bank (ATIB = 7)
  • ✓The fleet learns once, then builds bank #2 faster

Agents do the 80%. Humans own the 20%.

This split is not a slogan — it is the ISO 42001 human-oversight control, and the trust story that sells to banks. Each agent does the repetitive 80%; humans own the 20% that is accountability, relationships, judgement, and legal / regulatory / commercial sign-off.

GOVERNANCE · THE GOLDEN PATH

Every change walks the same path.

Agent or human, there is exactly one way a change reaches main. It is protected by rulesets on every repo — no direct push for anyone, not even admins or bots. Step through the flow below.

AGENT IDENTITY & RUNTIME

Agents propose. Humans dispose.

Agents don't get a person's login. They act as a GitHub App with its own identity — authoring pull requests, never impersonating a human, and never able to bypass the gate. The runtime toolkit that mints their credentials and opens their PRs lives in nova-agent-runtime/runtime/.

nova-agent-runtime/runtime
# 1 · mint a short-lived token for the GitHub App
./gh-app-auth.sh --repo nova-platform-backend
→ installation token (expires in ~1h)

# 2 · branch, commit, push & open the PR as the bot
./open-pr.sh --title "feat: add payout webhook" \
  --branch agent/payout-webhook
→ PR #128 opened by nova-ft-fleet-agent[bot]
→ verify CI running · awaiting human approval

FACTORY ECONOMICS

Platform stays flat. Only customers multiply.

This is the factory economics in repo form. The 12 platform repos are a fixed cost — they don't grow when a bank signs. Each new bank adds a fixed set of 7 customer repos. Drag the counter to feel it.

Twelve platform repos, once.

The agent fleet, the human surfaces, the portable banking platform and governance each evolve independently — but their count never changes as the business scales. Every bank after that is just +7, provisioned by the agents.
