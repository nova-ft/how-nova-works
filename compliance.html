AI SECURITY & COMPLIANCE · GRC-AS-CODE

Compliance, baked in.

Steered by Hussen & Hamza · Operations, Security & Compliance

For a bank, the certification set is the licence to sell. So Nova FT doesn't bolt compliance on at audit time — it runs GRC-as-code: controls, evidence, policies and audits live in the nova-compliance repo and are enforced as gates in the engineering SDLC. Every change is scanned, mapped to controls, and can't ship until it passes — evidence captured automatically, audited continuously by agents.

THE DEPARTMENT · TWO AGENTS

A Security agent and a Compliance agent.

The AI Security & Compliance function runs on two agents on the same earn-trust autonomy model as the rest of the fleet — tiers 0–3 autonomous or human-approved. The one thing neither does autonomously: attest a certification. That is tier 4 — a named human signs, always. It is itself an ISO 42001 human-oversight requirement.

Security agent
nova-agents/fleets/operations/security

The platform's watchdog on the wire and in the code. It scans every change and every deploy for weakness, and keeps a live picture of the threat surface.

  • ›SAST, secret & dependency scanning — CodeQL, push protection, Dependabot / dependency review.
  • ›Threat monitoring — continuous detection across the running platform.
  • ›Pen-test checklist — seeded from the ATIB security spec, run each release.
Autonomy 0–3 scan & report autonomously · propose fixes, human approves
Compliance agent
nova-agents/fleets/operations/compliance

The regulatory reviewer sitting beside the code reviewer. Its single job: keep every control in every framework green, continuously, with audit-ready evidence.

  • ›Continuous audit vs ISO 27001 / SOC 2 / 42001 / PCI — daily sweep, not a yearly check.
  • ›PR-gate — blocks any change that would turn a control red, before it ships.
  • ›Evidence collection & the live certification tracker — audit pack in minutes, not weeks.
Autonomy 0–3·4 audits & gates autonomously · a human attests (tier 4), never the agent
THE KEY IDEA · CONTROLS IN THE PIPELINE

Compliance is a gate, not an afterthought.

Every feature walks the same SDLC. Two stages — Security and Compliance — are where controls are enforced. Each automated check maps to a control and auto-produces evidence. A PR can't merge, and a change can't ship, until its controls pass. Click a gate, then a check, to trace check → control → evidence.

The Security and Compliance gates are highlighted & clickable — a feature can't pass them until its controls are green.

THE CONTINUOUS AUDIT LOOP

It never stops watching.

Controls drift the moment code ships — a new secret, an over-broad role, an unencrypted field. A quarterly human check can't keep a fast-moving AI-built platform green. So the agents run a loop: scan → map to controls → collect evidence → audit → flag gaps → raise issues on the Kanban — closing straight back into the engineering SDLC. Click a step.
